프로젝트 규칙

  • 컴포넌트명 : Kebab case 사용
  • class 속성 순서 : display > position > width height >spacing > bg > text(크기 >색상>두께)
  • 동일한 속성일 경우 브레이크포인트별로 묶어서 사용
  • 비슷한 속성끼리 묶어서 줄넘김
  • 단위는 rem 단위 사용
  • 🎉 Init: 초기 설정
  • 🎨 Feat: 기능 추가, 변경
  • 🔥 Remove : 파일이나 코드 삭제
  • 📝 Docs: readme.md, json 파일 등 수정, 라이브러리 설치
  • 💄 Style: CSS 등 사용자 UI 디자인 변경
  • ♻️ Refactor: 코드 리팩토링
  • 🧪 Test: 테스트 코드 추가, 삭제, 변경 등
  • 🐎 Ci: npm 모듈 설치 등
  • 🐳 Chore: 패키지 매니저 설정할 경우, 운영 코드 변경이 없는 경우
  • main: 최종 배포 브랜치
  • develop: 개발 브랜치
  • develop/feat/components: 컴포넌트 개발 브랜치
  • develop/pages: 페이지 개발 브랜치
  • develop/style: 스타일링 개발 브랜치
  • develop/fix: 개발 핫픽스 브랜치
  • develop/setting: 개발 환경 브랜치
